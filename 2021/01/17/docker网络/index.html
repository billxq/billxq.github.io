docker网络

none网络

挂在这个网络的容器，除了lo，没有其它任何网卡。可以通过–network=none来指定使用none网络。使用场景，如安全性相对较高的应用，可以使用none网络。可以让这个容器随机生成密码，就可以放到none网络，避免密码被窃取。

host网络

容器的网络配置与host完全一致。通过–network=host来指定容器使用host网络。对网络传输要求较高的容器可以选择host网络，但是要考虑端口冲突问题。

bridge网络

这是docker默认的网络方案，创建的容器默认会挂载到docker0上。可以通过docker network inspect bridge命令来查看bridge网络的配置信息。

user-defined网络

Docker提供三种user-defined网络驱动：bridge，overlay和macvlan。overlay和macvlan用于创建跨主机的网络。创建方法：docker network create --driver bridge --subnet 172.22.16.0/24 --gateway 172.22.16.1 my_net
